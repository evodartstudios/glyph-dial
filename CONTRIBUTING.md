# Contributing to Nothing Dialer

First off, thank you for considering contributing to Nothing Dialer! It's people like you that make the open source community such a fantastic place to learn, inspire, and create. This is EvoDART's first major open source push!

## How can I contribute?

### 1. Find an Issue
Check out our [`TODO.md`](./TODO.md) file! This serves as our master roadmap for features. Pick something that matches your skill level:
- Missing basic Dialer functions (Mute toggles, Output device switching, etc).
- Creating smooth Jetpack Compose UI/UX flows.
- Connecting deep Android permissions.

### 2. Fork & Setup
1. Fork the repo and clone it locally.
2. Open the project in **Android Studio Koala** or newer.
3. Sync Gradle and ensure the base app runs on your emulator/device.

### 3. Branches & Commits
- Create a branch for your feature. Example: `feature/multi-sim-support` or `fix/incall-notification`.
- Try to make your commits atomic and your commit messages descriptive. 

### 4. Create a Pull Request
1. Push your branch to your Fork.
2. Open a Pull Request against our `main` branch.
3. In your PR description, explain **what** you did, **why** you did it, and provide screenshots/videos if it involves UI changes.

## Code Style
- We use **Kotlin** and **Jetpack Compose**.
- Try to keep your UI logic declarative.
- Adhere to general Android/Kotlin standard conventions.

## Need help?
If you get stuck or want to discuss architectural decisions before making a massive PR, reach out to us at our Instagram/LinkedIn listed in the root `README.md`.

We look forward to building this with you!
