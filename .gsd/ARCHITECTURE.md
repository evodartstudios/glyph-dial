## Overview

GlyphDial is a **Jetpack Compose dialer app** designed for Nothing phones, structured as a single-module Android application using **MVVM** with **Hilt DI**. It acts as a full replacement default dialer via Android's `InCallService` API, handling call management, contacts, call history, and settings.

## Components

### Application Entry

- **`GlyphDialApp`** — Hilt `@HiltAndroidApp` entry point. Minimal — no crash reporting, analytics, or initialization logic yet.
- **`MainActivity`** — Primary UI host. Contains `GlyphDialContent` composable with swipeable pager navigation across 5 tabs: Dial, Recents, Contacts, Favorites, Settings. Manages permissions, default dialer request, and detail screen overlays.
- **`InCallActivity`** — Launched by `GlyphDialCallService` during calls. Shows over lock screen. Contains `InCallContent` composable that renders incoming/outgoing/active call screens based on call state. Performs contact lookup for caller ID.

### Service Layer

- **`GlyphDialCallService`** — Core `InCallService` implementation. Uses static `companion object` for call state management (`MutableStateFlow`). Handles call lifecycle (add/remove), notification channels (incoming/active), `CallStyle` notifications (Android 12+), audio routing, and mute/hold controls.
- **`CallActionReceiver`** — `BroadcastReceiver` for notification action buttons (Answer, Decline, Hang Up). Delegates to `GlyphDialCallService` static methods.
- **`CallState`** enum — Maps Android `Call.STATE_*` constants to app-level enum for UI consumption.

### Data Layer

#### Models
- **`Contact`** — Device contact with `id`, `name`, `phoneNumbers: List<PhoneNumber>`, `photoUri`, `starred`. Computed properties: `displayName`, `primaryNumber`, `initials`.
- **`PhoneNumber`** — Number with `type` (MOBILE/HOME/WORK/OTHER) and optional label.
- **`CallLogEntry`** — Call history entry with `id`, `number`, `name`, `type` (INCOMING/OUTGOING/MISSED/REJECTED/BLOCKED/VOICEMAIL), `timestamp`, `duration`, `photoUri`, `isRead`.

#### Repositories
- **`ContactRepository`** (`@Singleton`) — Reads device contacts via `ContentResolver`. Methods: `getAllContacts()`, `searchContacts()`, `getStarredContacts()`, `lookupContactByNumber()`. Uses `Flow<List<Contact>>` for reactive data. Phone number lookup handles country code normalization (India +91, US +1, UK +44).
- **`CallLogRepository`** (`@Singleton`) — Reads call history via `ContentResolver`. Methods: `getRecentCalls(limit)`, `getMissedCalls()`. Uses `Flow<List<CallLogEntry>>`.

#### Settings
- **`SettingsDataStore`** (`@Singleton`) — DataStore Preferences for: `defaultStartPage`, `scrollbarPosition` (LEFT/RIGHT), `showRecommendations`, `accentColor`.
- **`AccentColor`** enum — 8 accent color options (RED, BLUE, GREEN, ORANGE, PURPLE, CYAN, PINK, WHITE) with hex values.
- **`ScrollbarPosition`** enum — LEFT or RIGHT.

## Data Flow

1. **App launch** → `MainActivity` → permissions requested → `MainViewModel.onPermissionsGranted()`
2. **Contact/Call log loading** → `MainViewModel` calls repository `.collect()` → `StateFlow` updated → UI recomposes
3. **T9 search** → User types digits → `DialPadScreen` → `MainViewModel.updateSearchQuery()` → in-memory filter of loaded contacts → suggestions shown
4. **Making a call** → `ACTION_CALL` intent from any screen → System routes through `GlyphDialCallService.onCallAdded()` → launches `InCallActivity` → shows call notification
5. **Incoming call** → System triggers `GlyphDialCallService.onCallAdded()` → static `StateFlow` updated → `InCallActivity` renders `IncomingCallScreen`
6. **Settings changes** → User picks option → `SettingsDataStore.set*()` → DataStore persists → `Flow` emits → UI recomposes

### Architecture Issues
- [ ] **Single ViewModel anti-pattern** — `MainViewModel` handles contacts, call log, search, and permissions. Should be split into per-feature ViewModels
- [ ] **Static state in service** — `GlyphDialCallService` uses `companion object` with static `MutableStateFlow`. Works but is fragile; consider DI-injected state holder
- [ ] **Room declared but unused** — Room DB is in dependencies but no `@Entity`, `@Dao`, or `@Database` classes exist. Either use it (for caching, speed dial, blocked numbers) or remove it
- [ ] **Duplicated SearchBar** — Identical `SearchBar` composable exists in both `RecentsScreen.kt` and `ContactsScreen.kt`. Should be extracted to shared component
- [ ] **No error handling** — Repository `catch` blocks are empty (`{ e -> }`). No error states exposed to UI
- [ ] **No loading/error states in ViewModel** — Only `isLoading` exists. No error state, retry mechanism, or empty state differentiation
- [ ] **No NavHost integration** — Routes.kt defines routes but navigation uses custom pager. The `GlyphDialNavHost.kt` file exists but isn't wired up
- [ ] **No ProGuard rules** — R8 shrinking is enabled but `proguard-rules.pro` may lack keep rules for Hilt, Room, and serialized models
- [ ] **No unit tests** — Only template test files exist (`ExampleUnitTest.kt`, `ExampleInstrumentedTest.kt`)
- [ ] **Action constants duplicated** — `ACTION_ANSWER/DECLINE/HANG_UP` defined identically in both `GlyphDialCallService.kt` and `CallActionReceiver.kt`

### Performance Concerns
- [ ] **Contact loading N+1 query** — `getAllContacts()` queries contacts list, then makes a separate query per contact for phone numbers. Should use a single join query
- [ ] **No contact caching** — Contacts are loaded fresh every time from ContentResolver. Should cache in Room for faster startup
- [ ] **T9 search runs on main data set** — `updateSearchQuery()` filters `_contacts.value` in-memory. Fine for small lists, but could be slow for 1000+ contacts without indexing

## Conventions

**Naming**: PascalCase for classes/composables, camelCase for functions/properties. Files named after primary class. Screen files use `*Screen.kt` pattern.

**Structure**: Feature-based package organization under `ui/screens/`. Shared components under `ui/components/`. Theme under `ui/theme/`. Data layer cleanly separated into `model/`, `repository/`, `settings/`.

**Testing**: No meaningful tests. Template files only.

**Build**: Version catalog (`libs.versions.toml`), single module, compileSdk 36, minSdk 24, Java 17. R8 minification + resource shrinking enabled for release.
